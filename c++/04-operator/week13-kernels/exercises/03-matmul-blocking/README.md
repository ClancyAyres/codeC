# 习题 03：矩阵乘法分块优化

## 目标

实现矩阵乘法的三级优化（朴素 → 循环交换 → 分块），并使用 benchmark 对比性能。

## 要求

1. **实现 `matmulNaive`**：
   - 标准 i-j-k 三重循环
   - 先清零 C，再累加

2. **实现 `matmulIkj`**：
   - 使用 i-k-j 循环顺序
   - 将 A[i*K + k] 提取到局部变量，在内层 j 循环中复用

3. **实现 `matmulBlocked`**：
   - 选择合适的分块大小 BLOCK（推荐 32/64/128）
   - 外层 i0/k0/j0 以 BLOCK 步进
   - 内层在 BLOCK 范围内运行 ikj 顺序
   - 注意边界：`std::min(i0 + BLOCK, M)`

4. **性能对比**：
   - 使用已提供的 `benchmark()` 函数计时
   - 计算每种方法的 GFLOPS
   - 计算相对于朴素版本的加速比

5. **验证正确性**：
   - 使用 `verify()` 确认所有优化版本结果一致

## 提示

- 分块大小的选择：
  - BLOCK×BLOCK×3（A子块 + B子块 + C子块）× 8 字节 ≈ L1 cache 大小
  - 例如 BLOCK=64: 3×64²×8 = 98KB（接近 L1，实际可能装入 L2）
  - BLOCK=32: 3×32²×8 = 24KB（装入 L1 有余）

- 外层循环顺序建议：i0 → k0 → j0（让 C 子块在寄存器/L1 中最大复用）

- 测试不同 SIZE：256（快）、512（中等）、1024（慢但更明显）

## 预期结果

- ikj 版本比朴素快 3-5×
- 分块版本比朴素快 5-20×（取决于矩阵规模和 CPU 缓存层次）
- 所有版本的输出数值一致（在 1e-9 容差内）
